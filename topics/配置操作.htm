配置操作

读配置

读取配置，表头 补充 三个字段 "[配置]", "[激活]", "[改配置]"。 当前配置会写在 "[激活]" 列； 所有配置，逗号分开，会写在 "[配置]" 列。

改配置

修改配置，完成后会更新   "[配置]", "[激活]"， 这2个字段。

"[改配置]"  列 和 界面上 配置后面的输入框，为修改配置的输入选项。 优先级上 [改配置] > 输入框。 优先解析  [改配置]  列，每一行都有有自动特定的 修改选。  [改配置]  为为空值或者 无效值的时候才会 使用 输入框内的修改选项。

< H3 > 改配置规则说明:

1. 存在2种符号: -> 派生 ; = > 改名
2. ->或者 = > 前后写入要操作的配置，可以有空格。< BR > 3. 可以写多个，逗号分割。执行的时候是从前到后依次执行

改配置规则:

A->B,->B,A->,->  分表表示: <br/>
  0. A->B  派生配置 由A派生创建B，执行成功条件: A存在，B不存在  <br/>
  1. ->B   创建B ，执行成功条件: B不存在  <br/>
  2. A->  删除A。行的条件为: 有多个配置。只有A的话，则保留。 <br/>
  3. ->   删掉派生,只保留顶层<br/>
A=>B,=>B,A=>,=> <br/>
  4. A=>B 改名 A存在B不存在。 <br/>
  5. = >B 当前改名为B  执行成功条件: B不存在<br/>
  6. A=> A改为当前配置，原激活配置不动 <br/>
  7. => 只保留当前配置其他的删除。若当前不是最顶层的配置，则会找他的顶层配置，然后把其余的删除 <br/>
